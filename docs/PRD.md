## 1. 제품 개요

**삼바웨이브**는 무재고 위탁판매 셀러를 위한 **올인원 자동화 솔루션**이다.

16개 소싱처에서 상품을 자동 수집하고, AI가 상품명/이미지/카테고리를 가공한 뒤, 18개 마켓에 원클릭 등록하고, 주문/CS까지 통합 관리한다. 셀러가 수작업으로 하던 모든 과정을 자동화하여, **1인 셀러도 10만개 이상의 상품을 운영**할 수 있게 한다.

### 핵심 가치 제안

| # | 가치 | 설명 |
|---|------|------|
| 1 | **AI 자동화** | 상품명 가공, 이미지 필터링, 카테고리 매핑을 AI가 처리. 수작업 시간을 90% 이상 절감 |
| 2 | **다소싱처 통합** | 무신사, KREAM, 나이키 등 16개 소싱처를 한 곳에서 관리. 가격/재고 변동 실시간 반영 |
| 3 | **멀티마켓 원클릭 전송** | 스마트스토어, 쿠팡, 11번가 등 18개 마켓에 정책 기반 자동 등록. API 규격 차이를 솔루션이 흡수 |

### 현재 단계

- **Early Adopter**: 1~5명의 실제 셀러가 사용 중
- **SaaS 전환 진행 중**: 멀티테넌시 구조 구축 완료
- **듀얼 비즈니스 모델**: 월 구독 SaaS + 소스코드 판매

## 2. 문제 정의

### 무재고 위탁판매 셀러의 3대 고통점

**1) 수작업 과다**
- 소싱처에서 상품을 하나씩 복사해서 마켓에 등록
- 상품명 규칙, 이미지 가공, 카테고리 분류를 매번 수동 처리
- 가격 변동 시 수백~수만 개 상품을 하나씩 수정

**2) 멀티마켓 관리 복잡**
- 마켓마다 API 규격, 카테고리 체계, 상품명 규칙이 다름
- 계정별로 별도 로그인/관리 필요
- 주문이 마켓별로 분산되어 통합 관리 불가

**3) 소싱처 변동 실시간 반영 불가**
- 소싱처 가격이 변하면 마켓 판매가와 마진이 틀어짐
- 품절 상품을 팔아서 주문 취소 → 스토어 페널티
- 재고/가격 동기화를 자동으로 해주는 솔루션이 없음

### 기존 솔루션의 한계

| 솔루션 | 한계 |
|--------|------|
| **플레이오토** | 마켓 간 전송만 지원. 소싱처 수집/AI 가공/가격 정책 없음 |
| **사빌/셀링크** | 소싱처 제한적 (주로 도매꾹/오너클랜). AI 자동화 없음 |
| **수작업 (엑셀)** | 100개 이상이면 관리 불가. 가격/재고 동기화 불가능 |

**삼바웨이브가 해결하는 것**: 소싱 → 가공 → 등록 → 관리 전 과정을 하나의 파이프라인으로 자동화

## 3. 타겟 사용자

### 페르소나 A: 초보 셀러

| 항목 | 내용 |
|------|------|
| **프로필** | 위탁판매를 막 시작하려는 개인. 부업 또는 창업 준비 |
| **상품 규모** | 0~1,000개 |
| **마켓 수** | 1~2개 (스마트스토어, 쿠팡 위주) |
| **핵심 니즈** | "뭘 팔아야 하는지"부터 모름. 상품 수집~등록 자동화 필요 |
| **솔루션 가치** | AI 소싱으로 잘 팔리는 상품 추천 + 원클릭 등록으로 빠른 시작 |

### 페르소나 B: 중급 셀러

| 항목 | 내용 |
|------|------|
| **프로필** | 3개 이상 마켓에서 활발히 판매 중. 매출 성장 단계 |
| **상품 규모** | 1,000~10,000개 |
| **마켓 수** | 3~5개 |
| **핵심 니즈** | 마켓별 가격정책 관리, 재고 동기화, 주문 통합 처리 |
| **솔루션 가치** | 오토튠으로 가격/재고 자동 동기화 + 멀티마켓 통합 주문 관리 |

### 페르소나 C: 대량 셀러 / 사업자

| 항목 | 내용 |
|------|------|
| **프로필** | 법인 또는 전문 셀러. 직원 있음. 대규모 운영 |
| **상품 규모** | 10,000~100,000개 이상 |
| **마켓 수** | 5개 이상 (해외 마켓 포함) |
| **핵심 니즈** | 대량 상품 자동 관리, CS 자동화, 매출 분석, 자체 서버 운영 |
| **솔루션 가치** | 소스코드 구매로 자체 인프라 구축 + AI 대량 가공 + 워룸 모니터링 |

## 4. 핵심 비즈니스 플로우

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 상품수집  │ → │ 상품관리  │ → │ 정책생성  │ → │ 정책적용  │ → │ 카테고리  │ → │ 마켓전송  │ → │ 주문관리  │ → │ CS관리   │
│ (소싱)   │   │ (AI가공) │   │          │   │          │   │ (매핑)   │   │          │   │          │   │          │
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
```

### 각 단계 상세

| 단계 | 수동 작업 | 삼바웨이브 자동화 |
|------|-----------|------------------|
| **1. 상품수집** | 소싱처 사이트에서 상품을 하나씩 복사 | Chrome 확장앱이 16개 소싱처에서 자동 수집. AI가 브랜드×키워드 조합 추천 |
| **2. 상품관리** | 상품명 수정, 이미지 선별, 정보 정리 | AI가 상품명 가공, 이미지 필터링(CLIP), 금지어 자동 제거 |
| **3. 정책생성** | 엑셀로 마켓별 마진율 계산 | 마켓별 마진율/배송비/할인 공식을 정책으로 저장 |
| **4. 정책적용** | 상품마다 가격 수동 계산 | 정책 선택하면 전체 상품에 자동 적용. 가격대별 범위 마진 지원 |
| **5. 카테고리** | 마켓별 카테고리를 수동 매핑 | AI가 소싱처 카테고리를 마켓 카테고리로 자동 매핑 |
| **6. 마켓전송** | 마켓별 관리자에서 하나씩 등록 | 18개 마켓에 API로 일괄 전송. 상세페이지 HTML 자동 생성 |
| **7. 주문관리** | 마켓별로 로그인해서 주문 확인 | 모든 마켓 주문을 한 화면에서 통합 관리. 송장 자동 등록 |
| **8. CS관리** | 마켓별 고객 문의에 개별 답변 | 통합 CS 대시보드. 템플릿 답변. 톡톡 웹훅 연동 |

### 자동 동기화 (오토튠)

마켓에 등록된 상품은 **오토튠 시스템**이 지속적으로 모니터링:
- 소싱처 가격 변동 → 마켓 판매가 자동 재계산 + 재전송
- 소싱처 품절 → 마켓 판매중지 자동 처리
- 소싱처 재입고 → 마켓 판매 재개

### 5.2 상품 관리 (Product Management)

수집된 상품을 마켓 등록에 적합한 형태로 가공하는 시스템.

**주요 기능:**
- **AI 이미지 필터링**: CLIP ONNX 모델로 모델 컷/상품 컷 자동 분류
- **AI 태그/분류**: Gemini로 상품 태그 자동 생성, 카테고리 자동 분류
- **이미지 처리**: 배경 제거(RemBG), AI 착용컷 생성, R2 클라우드 저장
- **금지어 필터링**: 마켓별 금지어 자동 감지 및 제거
- **상품명 가공 규칙**: 브랜드명 위치, 키워드 추가/삭제 규칙 설정
- **무한스크롤**: 10만건 이상의 상품을 끊김 없이 로드 (`/collector/products/scroll`)

### 5.3 가격 정책 엔진 (Policy Engine)

마켓별 판매가를 자동 계산하는 정책 시스템.

**주요 기능:**
- **마켓별 정책**: 마켓마다 독립된 마진율/배송비/추가요금 설정
- **가격대별 범위 마진**: "1만원 이하는 마진 30%, 5만원 이상은 마진 15%" 등 구간 설정
- **통화 지원**: KRW, USD, JPY
- **과세/면세 구분**
- **오토튠 (자동 동기화)**:
  - 소싱처 가격 변동 → 마켓 판매가 자동 재계산
  - 소싱처 품절 → 마켓 판매중지
  - 가격/재고 분리 동기화
  - 병렬 2 + 인터벌 1초 속도 제어

### 5.5 주문/CS 관리

모든 마켓의 주문과 고객 서비스를 통합 관리하는 시스템.

**주문 관리:**
- 멀티마켓 주문 통합 조회 (주문접수~배송완료, 취소/반품/교환)
- 송장번호 입력 + 배송사별 자동 추적 링크 (CJ/한진/롯데택배 등)
- 플레이오토 주문 동기화 (주문확인/CS/반품교환)
- 수익률 자동 계산 (소싱가 대비 판매가)

**반품/교환:**
- 반품/교환/취소 요청 통합 관리

## 6. 시스템 아키텍처

### 3계층 구조

```
┌─────────────────────────┐
│    Chrome 확장앱 (v2)    │  ← 소싱처 쿠키 수집, KREAM CDP 수집, 잔액 동기화
│   extension/ & ext-web/ │
└────────────┬────────────┘
             │ REST API (쿠키 프록시, 수집 결과 전송)
             ▼
┌─────────────────────────┐
│   FastAPI 백엔드         │  ← 비즈니스 로직, 마켓 API 연동, AI 처리
│   Python 3.12 (async)   │
│   Cloud Run (2Gi)       │
└────────────┬────────────┘
             │ REST API (JWT 인증)
             ▼
┌─────────────────────────┐
│   Next.js 프론트엔드     │  ← 대시보드, 상품/주문/정책 관리 UI
│   React 19 + Tailwind   │
│   Vercel 배포           │
└─────────────────────────┘
```

### 플러그인 아키텍처

마켓/소싱처 연동은 **플러그인 패턴**으로 구현:

```
plugins/
├── markets/          ← MarketPlugin 구현체 (18개)
│   ├── smartstore.py
│   ├── coupang.py
│   └── ...
├── sourcing/         ← SourcingPlugin 구현체 (16개)
│   ├── musinsa.py
│   ├── kream.py
│   └── ...
├── market_base.py    ← 마켓 플러그인 추상 클래스
└── __init__.py       ← discover_plugins() 자동 탐색
```

새 마켓/소싱처를 추가할 때는 해당 디렉토리에 파일 하나만 추가하면 자동 등록.

### 기술 스택

| 영역 | 기술 |
|------|------|
| 백엔드 프레임워크 | FastAPI (async/await) |
| ORM | SQLModel + SQLAlchemy (asyncpg) |
| DB | PostgreSQL (Cloud SQL, Read/Write 분리) |
| 프론트엔드 | Next.js 15 (App Router), React 19 |
| 스타일링 | Tailwind CSS 4 + shadcn/ui |
| AI | Anthropic Claude (카테고리 매핑), Google Gemini (태그/이미지), CLIP ONNX (이미지 필터링) |
| 인증 | JWT (서버사이드 검증) |
| 파일 스토리지 | Cloudflare R2 |
| 캐시 | Redis (인메모리 폴백) |
| 배포 (백엔드) | Google Cloud Run (서울, 2Gi) |
| 배포 (프론트엔드) | Vercel |
| CI/CD | GitHub Actions |
| 패키지 관리 | uv (Python), pnpm (Node.js) |

### 도메인 구조 (DDD)

백엔드는 **도메인 주도 설계**로 구성. 각 도메인은 3-레이어:

```
domain/samba/{도메인}/
├── model.py        ← 데이터 모델 (SQLModel)
├── service.py      ← 비즈니스 로직
└── repository.py   ← 데이터 접근 (DB 쿼리)
```

주요 도메인 (27개): account, ai, ai_sourcing, analytics, cache, category, channel, collector, contact, cs_inquiry, emergency, forbidden, image, job, order, policy, probe, product, proxy, returns, shipment, sns_posting, sourcing_account, store_care, tenant, video, warroom, wholesale

### 삼바웨이브의 3대 차별점

**1. AI 자동화 (경쟁사에 없는 기능)**
- 상품명/이미지/카테고리를 AI가 자동 가공
- 사람이 하루 걸릴 작업을 분 단위로 처리
- Claude, Gemini, CLIP 등 최신 AI 모델 활용

**2. 소싱~판매 풀 파이프라인**
- 경쟁사는 "마켓 간 전송" 또는 "도매 소싱"만 지원
- 삼바웨이브는 소싱 → AI 가공 → 정책 → 전송 → 주문 → CS 전 과정 자동화
- 16개 소싱처 × 18개 마켓 = 288개 조합을 하나의 시스템에서 관리

**3. 소스코드 판매 (유일)**
- SaaS 구독 외에 소스코드 일괄 구매 옵션 제공
- 대형 사업자가 자체 서버에서 운영 가능
- 커스터마이징 자유도 제공
