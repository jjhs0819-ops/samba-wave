# 이슈보고 절차

버그를 발견하면 **직접 고치지 말고** 이 절차로 원작자에게 알린다.

## 1. 증상 수집

- 에러 메시지 원문 (그대로 복사, 요약 금지)
- 발생 시각, 발생 화면/작업
- 재현 가능 여부: 항상? 가끔? 특정 조건?
- 실제 결과 vs 기대 결과 (예: "알리고는 발송 성공인데 삼바는 실패로 기록")

## 2. 원인 분석 (코드는 읽기만)

- 관련 코드를 찾아 `파일경로:줄번호` 까지 특정한다.
- 왜 이런 동작이 나오는지 추적한다. 확정 못 하면 가능한 케이스를 나열한다.
- 같은 패턴이 다른 곳에도 있는지 확인한다 (영향 범위).

## 3. 이슈 작성 (아래 양식 고정 — 이슈 #1 형식)

```markdown
## 증상
- (사용자 관점에서 무엇이 잘못 보이는지)

## 원인
- (`파일:줄번호` 와 함께 코드 레벨 원인)

## 영향 범위
- (같은 패턴이 있는 다른 위치)

## 제안 수정
1. (구체적 수정 방향, 코드 강요 X)

## 검증 기준
- (고쳐졌는지 어떻게 확인할지)
```

## 4. 작성 후

- 이슈 제목: 한 줄로 증상 + 원인 요약 (예: "SMS 발송 성공인데 삼바에서 '발송 실패'로 기록됨 (JSON 파싱 오류)")
- 이슈를 올린 뒤 번호와 링크를 보고한다.
