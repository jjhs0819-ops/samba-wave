# 상품수집 절차

## 시작 전 확인

1. 오늘 수집 목표 확인: 어느 소싱처에서, 어떤 방식(URL / 키워드 / 수집필터 / 브랜드 갱신)으로, 몇 개나?
   - 사용자가 안 알려줬으면 먼저 물어본다.
2. 확장프로그램 로그인 상태 확인 (무신사 등 쿠키 기반 소싱처는 자동로그인 필요).

## 수집 방식별 절차

### A. URL 수집
1. 사용자가 준 URL 목록을 받는다.
2. 삼바 수집 화면(/collector)의 URL 수집으로 등록한다.
3. 수집 결과에서 실패한 URL이 있으면 원인(품절/차단/파싱실패)과 함께 따로 보고한다.

### B. 키워드/수집필터 수집
1. 기존 수집 필터를 재사용할지, 새 필터를 만들지 확인한다.
2. 새 필터라면: 키워드, 가격대, 제외 조건(프리오더 제외, 부티크 제외 등)을 정해서 생성.
3. 수집 실행 후 수집된 개수를 보고한다.

### C. 브랜드 갱신
1. 갱신 대상 브랜드 목록을 확인한다.
2. 브랜드 스캔/갱신 실행 → 신규 발견 상품 수를 보고한다.

## 수집 후 점검 (공통)

- [ ] 블랙리스트에 걸린 상품이 재수집되지 않았는지 확인
- [ ] 중복 수집 여부 확인
- [ ] 수집 결과 요약 보고: 소싱처별 수집 수 / 실패 수 / 다음 단계(가공) 대상 수
