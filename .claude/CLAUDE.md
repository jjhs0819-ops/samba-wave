- 끝까지 될 때까지 실행할 것. 정말 안 되는 것만 보고할 것 [중요]
- 텍스트가 2행에 걸치지 않도록 UI 수정
- 접근 실패한 방식은 반드시 기록에 남기도 동일한 형태로 시도하지 말것.
- 부분적인 성공도 기록을 남겨둘것
- 코드 수정전에 반드시 크롬창에서 계획한 방식으로 직접 실행하고 성공여부 검증할것
- 작은 수정이라도 확장앱 버전업데이드 반드시할것[중요]
- 마이그레이션 파일이 없어서 DB에 컬럼이 존재하지 않는지 반드시 확인[중요]
- 팝업창은 무조건 모달창으로 생성
- 코드수정시 검증을 위해 재시작해야하는 내용 반드시 보고 : 프런트, 백서버, 확장앱
- 코드 수정 전 해당 변경이 다른 곳에서 오류를 일으킬 가능성을 먼저 확인할 것 [중요]
- 스피너 사용하지 말것
- "삭제"는 마켓에서 상품을 완전히 삭제하는 것(마켓삭제)이며, 판매중지(SUSPENSION/STOP)와는 다른 개념이다. 혼동하지 말것 [중요]

## 브랜치 기반 파일 수정 제한 [중요]
- 현재 브랜치가 `feature/XXX-plugin`이면 → `plugins/markets/XXX.py`와 `proxy/XXX.py`만 수정
- 현재 브랜치가 `feature/XXX-sourcing`이면 → `plugins/sourcing/XXX.py`와 `proxy/XXX.py`(또는 `proxy/XXX_sourcing.py`)만 수정
- 위 파일 외 다른 파일 수정이 필요하면 → 수정하지 말고 "이 파일도 수정이 필요합니다. 팀장에게 확인하세요"라고 안내
- `__init__.py`, `market_base.py`, `sourcing_base.py`, `collector/` 폴더는 절대 수정 금지
