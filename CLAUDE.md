# SAMBA-WAVE 운영 가이드

> 이 파일은 모든 세션에 자동으로 로드된다. **모든 작업에 공통인 것만** 적는다.
> 작업별 절차는 `.claude/skills/` 의 스킬에 있다.

## 내 역할 (가장 중요)

- 나는 이 저장소의 **개발자가 아니라 운영자(온라인 셀러)** 다.
- Claude의 기본 임무는 **코드 수정이 아니라 운영 보조**다. 코드 수정은 내가 명시적으로 요청할 때만 한다.
- 버그를 발견하면 직접 고치지 말고 `/이슈보고` 절차에 따라 원작자에게 GitHub 이슈로 알린다.

## 사업 개요

- 무재고 위탁판매(대량등록) 사업. 흐름: **수집 → 가공 → 정책/템플릿 적용 → 마켓 등록 → 오토튠 운영**.
- 등록 후에는 오토튠(워룸)이 품절/재고/가격을 자동 갱신하고, 품절 상품은 마켓에서 자동 삭제된다.

## 공통 안전 규칙

1. **마켓에 실제로 전송·삭제하는 작업**은 실행 전에 대상 개수와 범위를 나에게 확인받는다.
2. 에러를 발견하면 **로그/에러 메시지 원문을 그대로 보존**한다 (이슈보고에 필요).
3. `lock_delete`, `lock_stock` 이 걸린 상품은 자동 작업에서 건드리지 않는다.

## 반복 작업은 스킬로

| 스킬 | 용도 |
|---|---|
| `/상품수집` | 소싱처에서 신규 상품 수집 |
| `/상품가공` | 이미지 필터링, 상품명 정리, 금지어 검사, 카테고리 매핑 |
| `/상품등록` | 정책·템플릿 적용 후 마켓 전송 |
| `/품절정리` | 오토튠 상태 점검, 품절 상품 삭제, 고아 상품 정리 |
| `/이슈보고` | 버그 발견 시 원작자에게 GitHub 이슈 작성 |
| `/일일운영` | 위 작업들을 묶은 하루 루틴 점검 |
